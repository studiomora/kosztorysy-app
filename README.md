# Kosztorysy — Brodacze z Lasu

Внутрішній застосунок для створення кошторисів (робота + матеріали + ПДВ) і генерації PDF у форматі,
максимально наближеному до існуючих кошторисів. Ціни підтягуються з власної бази (сформованої на
основі попередніх кошторисів), яку можна редагувати прямо в застосунку.

## Стек

- **Next.js 15** (App Router) + TypeScript + Tailwind CSS
- **Neon** (serverless Postgres) як база даних
- **Drizzle ORM** для схеми/запитів
- **@react-pdf/renderer** для генерації PDF (без headless-браузера — легко деплоїться на Vercel)
- Захист доступу простим спільним паролем (без окремого сервісу авторизації)

## Структура

```
src/
  app/
    login/            — сторінка входу (пароль)
    baza-cen/          — перегляд/редагування бази цін
    kosztorysy/        — список кошторисів, конструктор нового, перегляд, генерація PDF
  db/
    schema.ts          — таблиці: price_items, estimates, estimate_items
    seed-data.ts        — стартові дані бази цін (з 16 попередніх кошторисів)
  lib/
    pdf/EstimateDocument.tsx — шаблон PDF
```

## Що потрібно, щоб це запрацювало

### 1. Neon (база даних) — обов'язково

1. Зареєструйся на https://neon.tech (безкоштовний план достатній).
2. Створи проєкт (регіон — Europe, наприклад Frankfurt).
3. Скопіюй **Pooled connection string** (Dashboard → Connection Details → Pooled connection).
4. Встав його як `DATABASE_URL` (локально — у `.env`, на Vercel — у Project Settings → Environment Variables).

### 2. Пароль доступу — опційно, але рекомендовано

Онови `APP_PASSWORD` у `.env` (локально) і в Environment Variables на Vercel — це єдиний спільний
пароль для входу в застосунок. Якщо залишити порожнім — застосунок буде відкритий без пароля.

### 3. Лого фірми — опційно

Поклади файл `public/logo.png` і зміни `COMPANY.logoPath` у
`src/lib/pdf/EstimateDocument.tsx` на `"/logo.png"`, щоб лого з'явилося в шапці PDF. Реквізити фірми
(NIP, REGON, телефон) редагуються там само в об'єкті `COMPANY`.

## Локальний запуск

```bash
npm install
cp .env.example .env       # заповни DATABASE_URL і APP_PASSWORD
npm run db:push            # створює таблиці в Neon за схемою
npm run db:seed             # заповнює базу цін стартовими даними (~150 позицій)
npm run dev                 # http://localhost:3000
```

## Деплой на Vercel

1. Заливаєш цей код у свій репозиторій на GitHub.
2. На https://vercel.com → New Project → імпортуєш репозиторій (Next.js визначиться автоматично,
   налаштувань збірки міняти не треба).
3. У Project Settings → Environment Variables додаєш:
   - `DATABASE_URL` — той самий рядок з Neon
   - `APP_PASSWORD` — пароль доступу
4. Deploy. Після першого деплою один раз локально виконай `npm run db:push && npm run db:seed`
   (з тим самим `DATABASE_URL`, що і на Vercel), щоб створити таблиці й заповнити базу цін —
   це не робиться автоматично під час білда.

Домен за замовчуванням Vercel видасть сам (`*.vercel.app`); свій домен можна додати пізніше в
Project Settings → Domains.

## Як користуватись

- **База цін** (`/baza-cen`) — перегляд, пошук, додавання, редагування і видалення позицій.
  Розділено на 4 групи: Робота/Матеріали × PLN/EUR.
- **Кошториси** (`/kosztorysy`) — список усіх створених кошторисів.
- **Новий кошторис** (`/kosztorysy/new`) — заповнюєш дані клієнта, додаєш рядки в розділи
  "Роботи" і "Матеріали". Починаєш вводити назву позиції — з'являються підказки з бази цін;
  вибір підказки автоматично підставляє одиницю виміру і ціну (яку можна відредагувати).
  Суми рахуються наживо.
- Відкривши кошторис, тисни **"Завантажити PDF"** — документ згенерується на льоту в форматі
  Robocizna / Materiały / RAZEM, як у твоїх попередніх кошторисах.

## Можливі наступні кроки (не зроблено зараз, за потреби доробити)

- Автентифікація на кількох користувачів (зараз — один спільний пароль).
- Копіювання існуючого кошторису як шаблону для нового.
- Експорт бази цін у CSV/Excel і імпорт назад.
- Надсилання PDF клієнту на email прямо із застосунку (знадобиться сервіс типу Resend).
- Зберігання логотипу та реквізитів фірми через UI замість правки коду.

## Важливе застереження щодо цієї збірки

Через мережеві обмеження середовища, в якому цей код писався, **не було можливості виконати
`npm install` та `npm run build` для перевірки збірки наживо** (реєстр npm був недоступний).
Код написано вручну, уважно, на основі стандартних і стабільних патернів Next.js App Router,
Drizzle + Neon та @react-pdf/renderer — але перед першим деплоєм рекомендую прогнати
`npm install && npm run build` локально й показати мені помилки, якщо вони виникнуть, щоб я їх
одразу поправив.
